Semihosting

Overview

NuttX supports many types of semihosting (syslog, file host sharing, poweroff, etc). The focus of this document is syslog and file sharing, since these are the most common features used when porting NuttX to a new platform or when testing requires files from the host machine (i.e. to play an audio file).

Semihosting SYSLOG

This feature is highly valuable, particularly during the porting process to a new chip. It allows the developer to track events and diagnose issues during the early initialization phase, even if the UART driver support hasn’t been implemented. Furthermore, it serves as a critical debugging tool when a physical serial port isn’t available for monitoring.

These are the steps to get semihost syslog working on stm32f4discovery board, but it could be adapted to other boards as well:

  1. Select the stm32f4discovery board with the nsh profile:

$ ./tools/configure.sh stm32f4discovery:nsh
  Copy files
  Select CONFIG_HOST_LINUX=y
  Refreshing...

  1. Run the menuconfig to select the necessary options:

$ make menuconfig

  1. Enable the debug options to be displayed over syslog semihost (we want see the memory allocations) :

Build Setup  --->
    Debug Options  --->
        [*] Enable Debug Features
        [*]   Enable Error Output (NEW)
        [*]     Enable Warnings Output (NEW)
        [*]       Enable Informational Debug Output (NEW)
        ...
        [*]   Memory Manager Debug Features
        [*]     Memory Manager Error Output
        [*]     Memory Manager Warnings Output
        [*]     Memory Manager Informational Output

  1. Enable the semihost syslog support:

System Type  --->
    [*] Semihosting SYSLOG support

  1. We need to disable the /dev/console otherwise the serial initialization will be called:

RTOS Features  --->
    Files and I/O  --->
        [ ] Enable /dev/console (disable the /dev/console)

  1. Disable the serial driver and enable syslog buffer to speed up the output:

Device Drivers  --->
    [ ] Serial Driver Support  ---- (disable serial driver)

    System Logging  --->
        [*] Use buffered output

  1. Save and leave menuconfig.

  2. Compile the firmware:

$ make -j

  1. Flash the firmware in the board:

$ sudo openocd -f interface/stlink.cfg -f target/stm32f4x.cfg -c init -c "reset halt" -c "flash write_image erase nuttx.bin 0x08000000"

  1. Start the openocd server to wait for GDB connection:

$ sudo openocd -f interface/stlink.cfg -f target/stm32f4x.cfg -c init -c "reset halt"
